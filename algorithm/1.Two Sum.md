# [1. Two Sum](https://leetcode.cn/problems/two-sum/)

**[Easy]**

## Description

Given an array of integers `nums` and an integer `target`, return ***indices*** of the two numbers such that they add up to `target`.

You may assume that each input would have ***exactly* one solution**, and you may not use the *same* element twice.

You can return the answer in any order.



**Example 1:**

```
Input: nums = [2,7,11,15], target = 9
Output: [0,1]
Explanation: Because nums[0] + nums[1] == 9, we return [0, 1].
```

**Example 2:**

```
Input: nums = [3,2,4], target = 6
Output: [1,2]
```

**Example 3:**

```
Input: nums = [3,3], target = 6
Output: [0,1]
```

 

**Constraints:**

- `2 <= nums.length <= 104`
- `-109 <= nums[i] <= 109`
- `-109 <= target <= 109`
- **Only one valid answer exists.**

**Follow-up:** Can you come up with an algorithm that is less than `O(n2)` time complexity?

## Solution 1

Use two-layer for loop brute force computation.

## Solution 2

使用hash表构建，需要用到uhash.h头文件，[下载连接](https://github.com/troydhanson/uthash/archive/master.zip)。

leetcode解题页面：

~~~c
/**
 * Note: The returned array must be malloced, assume caller calls free().
 */


struct hashTable {
    int key; //存 num[i]
    int val; //存 i
    UT_hash_handle hh;
};

struct hashTable* hashtable;

struct hashTable* find(int ikey) {
    struct hashTable* tmp;
    HASH_FIND_INT(hashtable, &ikey, tmp);//查表
    return tmp;
}

void insert(int ikey, int ival) {
    struct hashTable* it = find(ikey); //是否已经在表中
    if (it == NULL) { //不在表中插入
        struct hashTable* tmp = malloc(sizeof(struct hashTable));
        tmp->key = ikey, tmp->val = ival;
        HASH_ADD_INT(hashtable, key, tmp);
    } else {
        it->val = ival; //在表中更新索引(无影响可以不更新)
    }
}

int* twoSum(int* nums, int numsSize, int target, int* returnSize) {
    hashtable = NULL;
    for (int i = 0; i < numsSize; i++) {
        struct hashTable* it = find(target - nums[i]); //先查表
        if (it != NULL) { //找到了直接返回下标数组
            int* ret = malloc(sizeof(int) * 2);
            ret[0] = it->val, ret[1] = i;
            *returnSize = 2;
            return ret;
        }
        insert(nums[i], i); //找不到插入继续循环
    }
    *returnSize = 0;
    return NULL;
}
~~~

你这段代码是典型用 **uthash 实现 Two Sum**，核心点就在这一句：

> **为什么要先 `find` 再 `insert`？**

我直接从“算法逻辑 + 哈希表语义 + bug规避”三层给你讲透。

------

### 一、核心原因（最关键一句话）

👉 **避免把当前元素和自己匹配（self-match）**

------

### 二、结合 Two Sum 逻辑来看

题目要求：

> 找到两个不同下标 i、j，使得
>  `nums[i] + nums[j] = target`

------

#### 当前代码逻辑（正确）

```
for (int i = 0; i < numsSize; i++) {
    find(target - nums[i]);  // 先查
    insert(nums[i], i);      // 再存
}
```

------

#### 举个具体例子

```
nums = [3, 3], target = 6
```

------

#### 正确流程（先 find）

| i    | nums[i] | 查找 target - nums[i] | hash 表内容 | 结果       |
| ---- | ------- | --------------------- | ----------- | ---------- |
| 0    | 3       | 查 3 → ❌              | {}          | 插入 3→0   |
| 1    | 3       | 查 3 → ✅              | {3→0}       | 返回 (0,1) |

✔ 正确答案

------

#### 如果你反过来（先 insert 再 find）

```
insert(nums[i], i);
find(target - nums[i]);
```

------

#### 错误流程

| i    | nums[i] | hash表 | 查找            | 结果         |
| ---- | ------- | ------ | --------------- | ------------ |
| 0    | 3       | {3→0}  | 查 3 → 命中自己 | ❌ 返回 (0,0) |

👉 **自己匹配自己，违反题意**

------

### 三、本质原因（更底层）

#### 1️⃣ 哈希表里存的是什么？

```
key = nums[i]
val = i
```

👉 key 是“值”，val 是“下标”

------

#### 2️⃣ find 的语义

```
find(target - nums[i])
```

👉 在找：

> 有没有“之前出现过”的数可以和当前 nums[i] 组成 target

------

#### 3️⃣ insert 的语义

```
insert(nums[i], i);
```

👉 把当前数登记进“历史记录”

------

#### 所以顺序必须是：

```
先查历史 → 再加入当前
```

而不是：

```
先加入当前 → 再查（会查到自己）
```
